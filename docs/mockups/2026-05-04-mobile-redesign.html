EVE — directory layouts (v2)

Cutting all the editorial fields (pitch / perfect / insider / crowd / vibe). The reliable content is what's in the legacy admin form: name, address, phone, url, specials, categories, notes, tags, plus Google enrichment (hours, price, lat/lng) when present.

Specials text is the editorial content. Notes is secondary. The detail page is a directory entry, not an article. Fallback art uses only the place name (or initials) — no corner labels, no badges, no ornaments. Palette and composition are both hashed from the place ID, so each place gets a stable, distinct look without depending on category metadata we may not have.

Fallback art — initials and names only

Each fallback is a single typographic element: either initials (composition B) or full name in caps sans (A) or italic serif (C). No corner labels, no badges, no ornaments — nothing that depends on metadata we may not have. Variety comes from palette + composition + the place name itself. Palette and composition are both picked deterministically from a hash of the place ID, so each place gets a stable look but neighbours don't collide.
